## 总览:18 态 per-song 状态树

```
[前置] 歌曲目录在本地工作区   ← 硬性二元 gate,缺 = 不能开工

Block 1 控制台错误清零(auto_check 全绿才算过)
  1.1 分工表完整性(扒曲信息 / 风格标签等填齐)
  1.2 文件夹命名 + 5 目录结构(分轨wav/总轨wav/midi/csv/混音工程原文件)
  1.3 各目录文件齐全度(Vocal_midi + Mix_midi 必需 + BG_midi 若有)
  1.4 文件命名归一化(含混音工程文件命名 {歌曲名}_{乐器}{音源序号}(_{轨道序号}).wav)
  1.5 WAV 物理格式 / 时长(96k/24bit/2ch、≥180s,不可 auto 修)
  1.6 CSV 简单格式(表头 / mm:ss / 列数 / 段落标签)
  1.7 复检(全部 auto_check 重跑,无残留)

Block 2 数据集内容检查(人耳/人眼)
  2.1 乐器音源对照表 vs 混音工程文件名一致性
  2.2 其他 CSV 语义复核
  2.3 混音台 session 1:分轨 + 总轨(4 项:对齐 / 命名↔内容 / 主唱静默杂音 / 通听)
  2.4 混音台 session 2:源文件 + 总轨(2 项:源文件命名↔内容 / 通听)
  2.5 MIDI vs WAV 对齐(仅 Vocal_midi + BG_midi;Mix_midi 不做对齐)
  2.6 渲染节奏(开 toggle → 听 → 关 toggle)
  2.7 渲染结构(开 toggle → 听 → 关 toggle)
  2.8 音频质量通听(扒谱 / 人声 / 混音整体)

Block 3 收尾(park,等百度云开放平台 API)
  3.1 上传到百度网盘
  3.2 上传分享链接到分工表
  3.3 标记"已验收"到分工表(precondition: Block 1 + 2 全 pass)
```

**关键态**(不可 skip):`1.7 复检` / `3.3 标记已验收`。其他态默认可 skip,实测后再收紧。

## 用语约定

**触发归属:**
- `agent` —— LLM 直接调,无人介入
- `agent+confirm` —— LLM 提议,UI 弹卡片让人勾选/批准才执行
- `hybrid` —— LLM 触发 UI 切到指定状态,人完成动作,人按 pass/fail/cancel 三态返回
- `human-only` —— LLM **不该有**对应工具,只在 check 卡里点名提示
- `状态机` —— Electron main 内编排,不是 LLM 工具

**底层调用** 命名按脑暴 §2 / §4:`fs.*` / `sheet.*` / `mix.*` / `playback.*` / `human.*` / `audit.*` / `fix.*` / `ui.*` / `meta.*`。

**操作 ID:** `G-XX`(全局/跨态)/ `1.x-XX`(Block 1 第 x 态)/ `2.x-XX` / `3.x-XX`。

## Block 1 控制台错误清零

**目标:** 把"自动可识别且可修复"的低层错误清空,留下的全是要人耳/人眼判断的项,推进到 Block 2。

## 设计观察(写完冒出来的)

打算回头沉到脑暴 §4 / §6 / §7。

1. **`mix.load_song(song, mode=...)`(2.3-01 / 2.4-01)** 是粒度合并的典型 —— 集合"打开窗 → 清轨 → 加多轨 → 开播",1 个工具替 5 步,呼应脑暴 §4.1。`mode` 参数枚举两种 session 配置,避免开两个 IPC。
2. **混音台 check 卡(2.3-05 / 2.4-02)** 一次问多件事 → 不拆多张卡 → 节省多轮 LLM round-trip。意味着 `request_human_check` 的 schema 需要支持 `items: list[CheckItem]` 而非单 string,**更新脑暴 §3 的草稿签名**。
3. **`human-only` 行**(2.3-03/04/07 / 2.2-02 / 2.5-03 / 2.6-03 / 3.1-01)是写 mcp_server 时**不该暴露**的明确划线。打 mark 备查。
4. **原子 Op 由 agent 自构造**(2026-05-08 修订):`DeleteOp / MoveOp / CreateDirOp / RenameOp` 不做 `fix.propose_*` 单独工具 —— LLM 直接输出 dict 塞 confirm 卡,跟人在 UI 里"右键改名 / 删 / 拖放"对等,自由度一致。`fix.propose_*` 只保留有"真实计算"的:`propose_rename_plan`(规则批量起点)+ `propose_csv_*`(diff 生成)。`fixers.execute_autofix_plan` 升级支持 union。
5. **3.1→3.2→3.3 三连**(上传 → 写链接 → 标已验收)park 中。等百度云开放平台 API 出来后,可能值得合并为一个 `finalize_acceptance(song, baidu_url)` 复合工具。
6. **2.2 CSV 语义复核** 是 Block 2 里最容易跳的 check —— 如果 1.6 把 syntax 错都扫干净,语义错率不高。`skip_if_no_csv_errors` 默认开关值得做,但默认值倾向 false(脑暴 §4.4 "硬约束")。
7. **`expected_items` 字段** 是这次新冒出的需求 —— 让 check 卡呈现结构化勾选项。回 §7 第 5 个 open question("`ui_state` 怎么 schema 化")。
8. **2.6 / 2.7 的 toggle 关闭副作用** 算入态退出动作(2.6-05 / 2.7-03),不是单独工具 —— 状态机 exit hook 触发,agent 不必显式调。
9. **2.8 是否合并进 2.3** 取决于实测:如果 2.3 的 4 项里"通听质量"已能等效覆盖整体感受,可砍掉 2.8。先留。
10. **Mix_midi 一致性** 仅查存在(1.3),不查对齐(2.5)—— 脑暴 2026-05-08 决定;如果未来需要可在 2.5 加可选项。
