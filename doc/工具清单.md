## 总览

**25 个工具,按通道分:**

- **MCP → sidecar**(数据/文件层,13):`fs.*`(1) / `sheet.*`(4) / `audit.*`(3) / `fix.*`(5)
- **IPC → renderer**(UI 控制层,8):`ui.*`(4) / `mix.*`(2) / `playback.*`(2)
- **in-process Electron main**(4):`human.*`(3) / `meta.*`(1)

> **设计:agent 自由 vs 强制 propose** —— `fix_propose_*` 只保留有"真实计算"的(规则驱动批量改名 / CSV diff 生成),原子的 RenameOp / DeleteOp / MoveOp / CreateDirOp 由 agent **直接构造** dict 塞给 `ui_show_confirm_card`。这跟人在 UI 里能"右键改名 / 右键删 / 拖放"对等,自由度一致;写盘那道闸由 confirm 卡 + 哈希校验兜底,安全没松动。

> ⚠ MCP / Anthropic API 工具名约束 `^[a-zA-Z0-9_-]{1,64}$`,**不允许点号**。文档里写 `audit.run_check` 是分组可读性写法,实际工具名是 `audit_run_check`。

**硬约束(脑暴 §4.4 / §5 / §8):**
- `reviewer_name` 永远不通过工具参数从 LLM 传入;`assignment_sheet.py` 内部读 config
- `_rows` 系列(`list_my_pending_rows` / `list_my_accepted_rows`)**不**进 MCP,仅 `/dev/*` 给 dev panel
- `mark_accepted` precondition:本 session review_log 中 Block 1+2 各态都有 `result=passed`
- `fix.execute_plan` precondition:`approved_ops` 哈希 = 上一张 confirm 卡批的(防 LLM 绕过 confirm)
- `playback.toggle_*(on=true)` precondition:对应 csv 已通过 1.6
- agent 不能写音频 / MIDI / 二进制;CSV/文本可写但走 confirm 卡 diff
- 分工表写只通过 `sheet.write_baidu_link` / `sheet.mark_accepted` 两个高层工具,不暴露通用 sheet 写

## 场景 → 工具

每态按"扫 / 修 / 定位 / check / ..."罗列工具组合。同一工具会在多态出现是正常的。

## 工具签名(reference)

每条按"签名 → 一句话行为 → failure / precondition"格式。详细类型 schema 由 FastMCP / Pydantic 从 type hint 自动生成,这里只列函数头。

### fs.*(MCP)

#### `fs_song_exists(song_name: str) -> {exists: bool, path: str | None}`
入口前置 gate:歌曲文件夹是否在本地工作区。
- failure: 无,异常情况返 `exists=false`

### sheet.*(MCP)

#### `sheet_list_my_pending() -> {songs: list[PendingSong]}`
列当前 reviewer 未验收的歌(精简版,**不含 _rows**)。
- `PendingSong = {row_index, song_name, owner}`
- failure: 腾讯文档挂 → `{ok: false, code: "SHEET_FETCH_FAILED", message}`

#### `sheet_get_song_meta(song_name: str) -> {meta: dict, missing_required_fields: list[str]}`
拉本歌 meta + 必填字段 schema 比对。LLM 不需要知道列号。
- failure: 同上

#### `sheet_write_baidu_link(song_name: str, url: str) -> {ok: bool, row_index: int}`
写百度云链接到分工表。
- failure: URL 校验挂 → `{ok: false, code: "INVALID_URL"}`

#### `sheet_mark_accepted(song_name: str) -> {ok: bool}`
标记"已验收"。
- **precondition**(硬):本 session review_log 中 Block 1+2 各态全 `passed`
- failure: 不满足 → `{ok: false, code: "NEEDS_HUMAN_VERIFY", message, suggestion}`

### audit.*(MCP)

#### `audit_run_check(song_path: str) -> {errors: list[CheckError], by_code: dict[str, int]}`
全量复检入口(覆盖 1.2 / 1.3 / 1.5 / 1.6 / 1.7 / 2.1 部分)。
- `CheckError` schema 见 `sidecar/errors.py`
- failure: 路径不存在 → 返空,不抛

#### `audit_collect_mix_proj_inventory(song_path: str) -> {instrument_table, mix_proj_files, mismatches}`
2.1 抽乐器音源对照表 vs 混音工程实际文件名 diff。
- failure: 数据源缺失(对照表 csv 没读到)→ `{ok: false, code: "INVENTORY_SOURCE_MISSING"}`

#### `audit_get_prior_review(song_name: str) -> {reviews: list[ReviewLogEntry]}`
跨 chat 查这首歌历史(知识库,不按 chat_id 过滤)。读 `cache/review_log.jsonl`。
- failure: 无,无记录返 `reviews=[]`

### fix.*(MCP,均 agent+confirm)

> propose_* 全部不写盘,只产 ops。写盘只走 `fix_execute_plan` 一道门。
> **原子 Op(RenameOp / DeleteOp / MoveOp / CreateDirOp)由 agent 自己构造**,跟人在 UI 里"右键改名 / 删 / 拖放"对等;LLM 直接输出 dict,塞给 `ui_show_confirm_card`。propose_* 只用于"真实计算"型(规则批量 / CSV diff 生成)。

**Op schema(LLM 自构造时遵循):**

```python
RenameOp     = {"type": "rename",     "src": str, "dst": str}
DeleteOp     = {"type": "delete",     "path": str}              # 文件 / 文件夹皆可,走 send2trash
MoveOp       = {"type": "move",       "src": str, "dst_dir": str}
CreateDirOp  = {"type": "create_dir", "path": str}
CsvDiffOp    = {"type": "csv_diff",   "path": str, "unified_diff": str}  # 由 propose_csv_* 产,LLM 不直接造
```

**白名单(在 `fix_execute_plan` 强校验,不在 propose 端):**
- 所有 src / dst / path 必须在当前工作区根目录下(绝对路径前缀匹配)
- 跨工作区路径拒绝 → `{ok: false, code: "PATH_OUTSIDE_WORKSPACE", path}`
- 二进制内容写仍不可(CsvDiffOp 仅限 csv/文本;DeleteOp 可删任何类型,对应人在 UI 能 delete 任何东西)

#### `fix_propose_rename_plan(song_path: str) -> {ops: list[RenameOp], conflicts: list[str]}`
1.2 / 1.4 **规则驱动**的批量改名建议(命名归一化 + 混音工程文件命名)。复用 `fixers.build_autofix_plan`。
- agent 拿到后**任选 / 修改 / 补充**,不是强制全收
- 用途:覆盖 80% 常见情况的批量起点,LLM 不必逐文件想

#### `fix_propose_csv_header_rewrite(path: str) -> {ops: list[CsvDiffOp]}`
1.6 改 CSV 表头到规范模板。生成 unified diff(LLM 自己写 diff 易错)。

#### `fix_propose_csv_time_zero_pad(path: str, line_nos: list[int]) -> {ops: list[CsvDiffOp]}`
1.6 mm:ss 零填充。

#### `fix_propose_csv_diff(path: str, intent: str) -> {ops: list[CsvDiffOp]}`
1.6 通用 CSV diff(列数 / 标签错误)。`intent` 字段由 LLM 给修改意图,内部生成 unified diff。
- ⚠ 太通用,实测后可能拆细(见待补)

#### `fix_execute_plan(approved_ops: list[Op]) -> {executed, errors, path_updates}`
执行上一张 confirm 卡批准的 ops(混合类型:Rename / Delete / Move / CreateDir / CsvDiff)。
- **precondition**(硬):`approved_ops` 哈希 = 最近一次 `ui_show_confirm_card` 返回的 approved_ops 哈希
- **路径白名单**校验(见上)
- failure: 哈希不匹配 → `{ok: false, code: "PLAN_HASH_MISMATCH"}`;路径越界 → `PATH_OUTSIDE_WORKSPACE`

### ui.*(IPC)

#### `ui_select_song(song_path: str) -> {ok: bool}`
G-02 文件树切歌 + 触发后台扫描。

#### `ui_show_confirm_card(card: ConfirmCard) -> {approved_ops, rejected_ops, cancelled: bool}`
1.x confirm 卡。card 含每个 op 的 src/dst/diff 可视化数据。**阻塞式**(同 human_check)。

#### `ui_locate_file(path: str) -> {ok: bool}`
1.3 文件树高亮 / 滚到指定文件。

#### `ui_open_file(path: str) -> {ok: bool, panel: str}`
按扩展名打开到对应编辑器(csv / midi / wav)。

### mix.*(IPC)

#### `mix_load_song(song_path: str, mode: Literal["stems_plus_master", "proj_files_plus_master"]) -> {ok, tracks_loaded}`
2.3 / 2.4 §4.1 任务粒度典型:打开窗 / 清轨 / 加多轨 / 准备播放,1 个工具替 5 步。

#### `mix_show() -> {ok: bool}`
把混音台窗口前置。

### playback.*(IPC,不暴露 seek)

#### `playback_toggle_beat_render(on: bool) -> {ok: bool}`
2.6 节奏渲染开关。
- **precondition**(`on=true`):当前总轨已打开 + 对应 Beat.csv 已通过 1.6
- failure: → `{ok: false, code: "BEAT_CSV_NOT_READY", suggestion}`

#### `playback_toggle_structure_render(on: bool) -> {ok: bool}`
2.7 结构渲染开关。
- **precondition** / failure:同上(对应 Structure.csv)

### human.*(in-process)

#### `human_check(state: str, reason: str, ui_state: UiState, items: list[CheckItem]) -> {result, items, note}`
**唯一** human-in-the-loop 阻塞工具(见脑暴 §3 升级 schema)。
- `result`: `"pass" | "fail" | "cancel"`
- `items`: `list[CheckItemResult]` 逐项结果,fail 时知道哪几项挂
- `cancel`:整个 workflow 中止(脑暴 §3 语义,不只是本次 check)
- `items` 长度 1 退化为单项卡;UI 渲染可特化为两按钮 pass/fail

#### `human_notify(message: str) -> {ok: bool}`
G-11 / 1.5 给用户发提示文字(走 chat 输出),agent 不直接发外部消息。

#### `human_notify_conflict(state: str, log_says: str, reality_says: str) -> {decision: "redo" | "continue" | "cancel"}`
G-08 resume 对账不一致时阻塞 ask。

### meta.*(in-process)

#### `meta_escalate_to_human(reason: str) -> {acknowledged: bool}`
脑暴 §4.6 escape hatch:agent 卡死 / 反复出错时调,流程暂停等人接管。

> **全局 cancel** 不是工具(脑暴 §4.7):是 Electron 层中断信号,任意时刻打断当前 tool call。

## 实现优先级

1. **MCP server 骨架** + `audit_run_check` / `sheet_list_my_pending` / `sheet_get_song_meta` —— 跑通最小读路径 ✅(目前已落 2 个,sheet_get_song_meta 待加)
2. `fix_propose_*` 4 个(rename_plan + csv_header / csv_time / csv_diff)+ `fix_execute_plan` —— Block 1 写路径,扩 `fixers.execute_autofix_plan` 支持 RenameOp/DeleteOp/MoveOp/CreateDirOp/CsvDiffOp union
3. `human_check` IPC + 阻塞机制(脑暴 §3 实现)—— Block 1 单项 check 卡先跑通
4. `ui_*` 4 个 IPC + confirm 卡机制 —— UI 编排基建
5. `mix_load_song` / `playback_toggle_*` —— Block 2 听感类
6. `sheet_write_baidu_link` / `sheet_mark_accepted` —— Block 3 收尾(实测后再上)
7. `meta_escalate_to_human` / `human_notify_conflict` —— escape hatch / resume 对账
